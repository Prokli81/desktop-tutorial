# MyFitClub: регистрация и вход

Этот документ описывает шаг 2 проекта - регистрацию, вход и пригласительные
коды.

## Что работает в текущем прототипе

В статической версии уже есть локальный демо-flow:

1. Пользователь выбирает **Регистрация**.
2. Вводит имя, email, пароль и пригласительный код.
3. Приложение проверяет код.
4. По коду назначается роль:
   - `CLIENT2026` -> клиент;
   - `TRAINER2026` -> тренер;
   - `ADMIN2026` -> администратор.
5. Аккаунт сохраняется в браузере.
6. После выхода пользователь может выбрать **Вход** и зайти по email/паролю.

Также есть готовый демо-аккаунт:

- email: `anna@myfitclub.demo`;
- пароль: `fitclub`;
- роль: клиент.

## Важное ограничение

Сейчас аккаунты и пароли сохраняются только в `localStorage` браузера. Это
подходит для прототипа, но не подходит для настоящего продукта.

В production-версии нужно заменить локальное хранение на:

- Firebase Authentication или собственный backend;
- безопасное хранение паролей;
- серверную проверку пригласительных кодов;
- правила доступа по ролям.

## Как должен работать настоящий вариант

### Регистрация

1. Пользователь вводит email/телефон и пароль.
2. Пользователь вводит пригласительный код.
3. Backend проверяет:
   - существует ли код;
   - активен ли код;
   - не превышен ли лимит использований;
   - какую роль должен получить пользователь.
4. Создается аккаунт.
5. Пользователь получает роль.
6. Использование кода увеличивается на 1.

### Вход

1. Пользователь вводит email/телефон и пароль.
2. Система проверяет данные.
3. Если пользователь активен, он попадает в приложение.
4. Если пользователь заблокирован, вход запрещен.

### Выход

1. Пользователь нажимает **Выйти**.
2. Сессия очищается.
3. Пользователь возвращается на экран входа.

## Следующий технический шаг

Подключить реальную систему пользователей. Самый простой путь:

1. Firebase Authentication для email/password.
2. Firestore collection `InvitationCodes`.
3. Firestore collection `Users`.
4. Cloud Function или защищенная backend-логика для проверки кода.

Без backend-валидации код можно будет обойти, поэтому проверка пригласительного
кода должна выполняться не только на клиенте.
