# MyFitClub: план базы данных

Шаг 3 проекта - вынести данные из временной логики экрана в понятные
коллекции. В текущем прототипе это сделано через `data-store.js` и
`localStorage`, чтобы структуру можно было быстро заменить на Firestore или
backend API.

## Что уже имитируется в прототипе

Локальная база хранится в ключе `myfitclub:db:v1` и содержит коллекции:

- `users`;
- `invitationCodes`;
- `clubMessages`;
- `bookings`;
- `notifications`.

На экране профиля есть блок **Локальная база прототипа**. Он показывает
количество записей в основных коллекциях.

## Коллекции

### users

Назначение: аккаунты клиентов, тренеров и администраторов.

Поля:

- `id`;
- `name`;
- `email`;
- `password` - только в прототипе, в настоящем продукте не хранить так;
- `code`;
- `role`;
- `roleName`;
- `label`;
- `createdAt`.

### invitationCodes

Назначение: контроль закрытого доступа в приложение.

Поля:

- `code`;
- `role`;
- `roleName`;
- `isActive`;
- `usageLimit`;
- `usedCount`.

### clubMessages

Назначение: общий чат клуба.

Поля:

- `id`;
- `author`;
- `text`;
- `role`;
- `userId`;
- `createdAt`.

### bookings

Назначение: запись пользователя на занятие.

Поля:

- `id`;
- `userId`;
- `scheduleId`;
- `createdAt`.

### notifications

Назначение: важные уведомления клуба.

Поля:

- `id`;
- `time`;
- `title`;
- `text`;
- `isRead`.

## Как перенести на Firebase / Firestore

1. Создать Firebase project.
2. Подключить Firebase Authentication.
3. Создать Firestore collections:
   - `users`;
   - `invitationCodes`;
   - `clubMessages`;
   - `bookings`;
   - `notifications`;
   - позже `groups`, `directMessages`, `scheduleEvents`.
4. Заменить функции `MyFitClubStore.list`, `add`, `replace`, `update`, `count`
   на Firestore-запросы.
5. Проверку пригласительного кода перенести на защищенную серверную логику.
6. Для сообщений подключить realtime listeners.

## Важное правило безопасности

В прототипе пароль хранится локально только для демонстрации. В настоящем
продукте пароль не должен сохраняться в базе как обычный текст. За это должна
отвечать система авторизации: Firebase Authentication или backend с безопасным
хешированием.

## Следующий шаг после локальной базы

Следующий продуктовый шаг - **реальные чаты**:

- сохранять каждое сообщение в базе;
- получать новые сообщения без обновления страницы;
- разделить общий чат, группы и личные сообщения;
- добавить права доступа по роли.
