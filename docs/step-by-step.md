# MyFitClub: идем по порядку

Этот документ фиксирует последовательность работ, чтобы проект развивался без
хаоса.

## Шаг 1. Дизайн и интерактивный прототип

Цель: показать идею приложения без сложной разработки.

Что должно быть понятно:

- как человек входит по коду;
- какие роли есть в клубе;
- где общий чат;
- где личные сообщения;
- где группы;
- где расписание;
- как выглядит запись на занятие;
- где уведомления;
- что видит администратор.

Статус: начато в текущем прототипе.

## Шаг 2. Настоящая регистрация

Цель: заменить демо-вход реальными аккаунтами.

Нужно сделать:

- регистрацию по email или телефону;
- вход пользователя;
- восстановление доступа;
- проверку пригласительного кода;
- автоматическое назначение роли по коду.

Рекомендуемый старт: Firebase Authentication.

## Шаг 3. База данных

Цель: чтобы данные сохранялись не только в браузере.

Нужно сохранить:

- пользователей;
- пригласительные коды;
- чаты;
- сообщения;
- группы;
- расписание;
- записи на занятия;
- уведомления.

Рекомендуемый старт: Firestore.

## Шаг 4. Реальные чаты

Цель: чтобы участники могли переписываться вживую.

Нужно сделать:

- общий чат клуба;
- личные сообщения клиент-тренер;
- групповые чаты;
- отметку прочитанных сообщений;
- ограничение доступа по роли.

## Шаг 5. Админ-панель

Цель: чтобы владелец клуба управлял приложением без разработчика.

Нужно сделать:

- создание пригласительных кодов;
- управление пользователями;
- смену ролей;
- блокировку пользователей;
- редактирование расписания;
- публикацию объявлений.

## Шаг 6. Мобильная упаковка

Цель: превратить прототип в приложение для телефона.

Варианты:

- PWA: быстрее, открывается как приложение с сайта;
- FlutterFlow + Firebase: быстрый путь к iOS/Android;
- Flutter или React Native: больше контроля, но больше разработки.

## Шаг 7. Дополнительные фитнес-функции

Добавлять после первой рабочей версии:

- дневник тренировок;
- фото прогресса;
- питание;
- челленджи;
- оплата абонемента;
- видеоуроки;
- интеграция с Apple Health / Google Fit.

## Самый правильный следующий шаг

После текущего прототипа следующий технический шаг - **настоящая регистрация и
пригласительные коды**, потому что без этого приложение не станет закрытым
пространством для клуба.
