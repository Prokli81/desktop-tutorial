# MyFitClub: продуктовый план

MyFitClub - закрытое мобильное приложение для фитнес-клуба. Его первая цель -
дать клиентам, тренерам и администратору одно приватное место для общения,
расписания и клубных объявлений.

## 1. Для кого

- фитнес-клуб;
- студия йоги или пилатеса;
- тренерская команда;
- закрытое фитнес-сообщество;
- клиенты, которым нужен простой доступ к расписанию и тренеру.

## 2. Роли

### Клиент

Может:

- войти по пригласительному коду;
- видеть расписание;
- записываться на занятия;
- писать в общий чат;
- писать тренеру;
- читать уведомления клуба;
- участвовать в групповых чатах.

### Тренер

Может:

- отвечать клиентам;
- писать объявления;
- вести тематические группы;
- видеть записи на свои занятия;
- отправлять рекомендации.

### Администратор

Может:

- создавать пригласительные коды;
- назначать роли;
- управлять пользователями;
- редактировать расписание;
- модерировать чаты;
- отправлять важные уведомления.

## 3. Первая версия MVP

Обязательные функции:

1. Регистрация и вход.
2. Проверка пригласительного кода.
3. Профиль пользователя.
4. Общий чат клуба.
5. Личные сообщения клиент-тренер.
6. Групповые чаты по темам.
7. Расписание занятий.
8. Запись на занятие.
9. Уведомления.
10. Простая админ-панель.

## 4. Основные экраны

- приветствие;
- вход / регистрация;
- ввод пригласительного кода;
- главная;
- общий чат;
- личные сообщения;
- группы;
- расписание;
- уведомления;
- профиль;
- админ-панель.

## 5. Данные

### Users

- id;
- name;
- email или phone;
- photoUrl;
- role: `client`, `trainer`, `admin`;
- fitnessGoal;
- status: `active`, `blocked`;
- createdAt.

### InvitationCodes

- id;
- code;
- role;
- isActive;
- usageLimit;
- usedCount;
- createdBy;
- createdAt.

### Chats

- id;
- type: `club`, `direct`, `group`;
- title;
- participantIds;
- createdAt;
- lastMessageAt.

### Messages

- id;
- chatId;
- senderId;
- text;
- imageUrl;
- createdAt;
- readBy.

### ScheduleEvents

- id;
- title;
- trainerId;
- startsAt;
- endsAt;
- location;
- description;
- capacity;
- bookedUserIds.

### Notifications

- id;
- title;
- text;
- audience;
- createdAt;
- sentAt;
- isRead.

## 6. Рекомендуемый стек

Для быстрого старта:

- FlutterFlow - сборка мобильного приложения;
- Firebase Authentication - вход;
- Firestore - база данных;
- Firebase Storage - фото;
- Firebase Cloud Messaging - push-уведомления.

Для полноценной разработки с кодом:

- Flutter или React Native;
- Node.js/NestJS backend;
- PostgreSQL;
- WebSocket для чатов;
- S3-совместимое хранилище для файлов.

## 8. Критерий готовности первой версии

Первая версия готова, когда приглашенный пользователь может:

1. Войти в приложение.
2. Увидеть роль и профиль.
3. Прочитать объявления.
4. Написать сообщение.
5. Открыть группу.
6. Посмотреть расписание.
7. Записаться на занятие.
8. Получить уведомление.

Администратор при этом должен уметь управлять доступом, ролями и расписанием.
