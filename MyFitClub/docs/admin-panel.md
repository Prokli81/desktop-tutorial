# MyFitClub: админ-панель

Шаг 5 проекта - дать администратору управление клубом без изменения кода.
В текущем прототипе админ-панель работает локально через `data-store.js`.

## Что работает сейчас

Администратор может:

- видеть количество активных кодов;
- видеть количество пользователей;
- видеть количество занятий;
- создавать пригласительные коды;
- выбирать роль для нового кода: клиент, тренер, администратор;
- задавать лимит использования кода;
- видеть список пользователей;
- создавать новые групповые чаты;
- добавлять занятия в расписание.

## Как проверить

1. Зарегистрироваться с кодом `ADMIN2026` или войти админским аккаунтом, если он
   уже создан.
2. Открыть раздел **Профиль**.
3. Внизу появится блок **Админ-панель**.
4. Создать новый код, группу или занятие.
5. Проверить, что данные появились в списках и счетчиках.

## Коллекции, которые использует админка

- `invitationCodes`;
- `users`;
- `chats`;
- `messages`;
- `scheduleEvents`.

## Что нужно сделать в production

В настоящем приложении все действия админки должны проверяться на сервере:

- только пользователь с ролью `admin` может создавать коды;
- код нельзя создать дважды;
- лимит использования должен проверяться при регистрации;
- создание групп и занятий должно писаться в backend/Firebase;
- действия администратора желательно логировать.
