# MyFitClub: идем по порядку

Этот документ фиксирует последовательность работ, чтобы проект развивался без
хаоса.

## Шаг 1. Дизайн и интерактивный прототип

Цель: показать идею приложения без сложной разработки.

Статус: **готово** — экраны, онбординг, дизайн-система, GitHub Pages.

## Шаг 2. Настоящая регистрация

Цель: реальные аккаунты вместо демо-only.

Статус: **готово в коде** — Firebase Authentication, инструкция
`docs/firebase-setup.md`. Локальный режим сохранён как запасной.

## Шаг 3. База данных

Цель: общие данные для всех участников.

Статус: **готово в коде** — Firestore для чатов, сообщений, расписания,
записей. Инструкция `docs/firestore-setup.md`.

## Шаг 4. Реальные чаты

Цель: переписка вживую между участниками.

Статус: **готово** — realtime, непрочитанные, браузерные и FCM push (после vapidKey + Cloud Function).

## Шаг 5. Админ-панель

Цель: владелец клуба управляет без разработчика.

Статус: **расширено** — коды, группы, расписание, объявления в чат, блокировка пользователей.

## Шаг 6. Мобильная упаковка

Цель: приложение на телефоне.

Статус: **в процессе** — PWA manifest, service worker, плашка установки.
Инструкция `docs/pwa-install.md`. Следующее: FCM, магазины приложений.

## Шаг 7. Дополнительные фитнес-функции

Статус: **в процессе** — дневник тренировок в облаке (вкладка «Дневник»),
восстановление пароля по email, смена пароля в профиле.

Дальше: фото прогресса, питание, оплата, видеоуроки, интеграции Health.

## Шаг 8. Email-авторизация (без телефона)

Статус: **готово** — регистрация и вход по email, «Забыли пароль?»,
смена пароля в профиле. Инструкции: `PROSTO-RU.md`, `email-spam-RU.md`.

Телефон/SMS — отложено.

## Текущий фокус

**Тест с вторым пользователем** (`docs/invite-friend-RU.md`), затем **FCM push**
(`docs/fcm-setup.md`).
