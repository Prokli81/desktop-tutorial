# MyFitClub: реальные чаты в прототипе

Шаг 4 проекта - превратить статические блоки общения в единую модель чатов.
В текущем прототипе это сделано локально через коллекции `chats` и `messages`.

## Что работает сейчас

- Общий чат клуба использует `chatId = club-main`.
- Групповые чаты открываются из раздела **Группы**.
- Личные диалоги открываются из раздела **Личные**.
- У каждого чата есть отдельная история сообщений.
- Новое сообщение сохраняется в коллекцию `messages`.
- Счетчик `Messages` в профиле обновляется после отправки.

## Коллекция chats

Поля:

- `id`;
- `title`;
- `type`: `club`, `group`, `direct`;
- `description`;
- `participantIds`;
- `createdAt`.

Примеры:

- `club-main` - общий чат клуба;
- `group-nutrition` - группа "Питание";
- `direct-maria` - личный диалог с тренером Марией.

## Коллекция messages

Поля:

- `id`;
- `chatId`;
- `author`;
- `text`;
- `role`;
- `userId`;
- `createdAt`.

## Как это перейти на настоящий backend

1. Хранить `chats` и `messages` в Firestore или backend базе.
2. Для каждого открытого чата подключать realtime listener.
3. Ограничить доступ:
   - `club` доступен всем активным участникам;
   - `group` доступен участникам группы;
   - `direct` доступен только двум сторонам и администратору при необходимости.
4. Добавить `readBy` для статуса прочтения.
5. Добавить вложения: фото, голосовые, файлы.

## Следующий шаг

После локальных чатов логичный следующий шаг - **админ-панель**:

- создавать группы;
- создавать пригласительные коды;
